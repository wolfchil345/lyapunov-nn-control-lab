# Internationalization Guide

This project supports viewer-facing documentation in four languages:

- English
- Japanese
- Korean
- Thai

## Translation policy

Translate viewer-facing content such as README files, documentation, release notes, contribution guides, issue templates, and pull request templates.

Do not translate Python function names, file names used by code, workflow syntax, package configuration, or command names.

## Folder structure

```text
docs/en/
docs/ja/
docs/ko/
docs/th/
```

Each translated document should keep the same file name in every language folder.

## Language switcher rule

Every translated document should start with language links so readers can change languages easily.

## Release rule

Create the final portfolio release only after core viewer-facing documents are available in all four languages.
