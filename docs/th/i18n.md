# คู่มือการทำหลายภาษา

โปรเจกต์นี้ตั้งเป้าให้เอกสารสำหรับผู้อ่านสามารถแสดงได้ 4 ภาษา

- English
- 日本語
- 한국어
- ไทย

## นโยบายการแปล

แปลเนื้อหาที่ผู้อ่านเห็น เช่น README เอกสาร release notes คู่มือการมีส่วนร่วม issue templates และ pull request templates

ไม่แปลชื่อฟังก์ชัน Python ชื่อไฟล์ที่โค้ดใช้งาน syntax ของ workflow การตั้งค่าแพ็กเกจ หรือชื่อคำสั่ง

## โครงสร้างโฟลเดอร์

```text
docs/en/
docs/ja/
docs/ko/
docs/th/
```

เอกสารที่แปลแล้วควรใช้ชื่อไฟล์เดียวกันในทุกโฟลเดอร์ภาษา

## กฎการเปลี่ยนภาษา

ทุกเอกสารที่แปลแล้วควรมีลิงก์เลือกภาษาไว้ด้านบน

## กฎก่อน release

สร้าง release สำหรับ portfolio หลังจากเอกสารหลักสำหรับผู้อ่านมีครบทั้ง 4 ภาษาแล้ว
