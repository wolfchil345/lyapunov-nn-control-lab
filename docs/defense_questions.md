# Defense Questions

This document lists possible questions and answer points for presenting or defending the Lyapunov Neural-Network Control Lab.

## Project motivation

### Why use a neural-network controller?
- Neural networks can approximate flexible control policies.
- They are useful when classical controller design becomes difficult for nonlinear systems.
- This project studies them in a simple system first.

### Why compare with LQR?
- LQR is a reliable classical baseline for linear systems.
- It provides a clear teacher controller for imitation learning.
- Comparing with LQR makes the neural controller easier to evaluate.

## Stability

### Why use Lyapunov-style checks?
- Stability is important in control engineering.
- Lyapunov analysis gives a way to reason about whether system energy-like behavior decreases.
- The project uses grid-based checks as empirical stability evidence.

### Does this prove global stability?
- No.
- The grid check only evaluates sampled states.
- It supports analysis, but it is not a full mathematical proof.

## Experiments

### Why use a mass-spring-damper system?
- It is simple, understandable, and common in control education.
- It has position and velocity states, which makes it easy to visualize.
- It is a good first testbed before moving to harder nonlinear systems.

### Why test robustness?
- Real systems have noise, actuator limits, and parameter uncertainty.
- Robustness experiments show whether the controller still works under selected imperfect conditions.

## Neural-network training

### What does the neural network learn?
- It learns a mapping from state to control input.
- The target control input is generated by the LQR controller.

### Why add a stability-aware penalty?
- Pure imitation may match LQR outputs but still behave poorly in closed-loop simulation.
- The penalty encourages better Lyapunov-style behavior around sampled states.

## Results interpretation

### Which metric is most important?
- No single metric is enough.
- Final state norm, settling time, cost, control energy, Lyapunov behavior, and robustness should be interpreted together.

### What result would be considered successful?
- The neural controller should converge near the origin.
- It should perform close to LQR.
- It should show mostly negative Lyapunov derivative values in the checked region.
- It should remain reasonable under noise, saturation, and parameter variation.

## Limitations

### What are the main limitations?
- The plant is simple.
- Simulations are not hardware experiments.
- Grid-based stability checks are empirical.
- The neural network may fail outside the training region.

## Future work

### How can this become stronger research?
- Test nonlinear plants.
- Add formal verification.
- Compare with KAN-based controllers.
- Learn neural Lyapunov functions directly.
- Apply the method to more realistic control problems.
